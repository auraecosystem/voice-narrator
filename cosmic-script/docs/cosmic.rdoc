## What KerboScript actually gives you (important)

## Step 4: Your runtime (VERY SIMPLE ENGINE)

This is your first interpreter:

```ts
class CosmicKernel {
  variables: Record<string, any> = {};

  execute(script: string) {
    const lines = script.split("\n").map(l => l.trim());

    for (let line of lines) {
      const [cmd, ...rest] = line.split(" ");

      switch (cmd) {
        case "PRINT":
          console.log(rest.join(" ").replace(/"/g, ""));
          break;

        case "SET":
          const [name, , value] = rest;
          this.variables[name] = Number(value) || value;
          break;

        default:
          console.log("Unknown:", line);
      }
    }
  }
}
```

This code is a basic example of **WebAuthn (Web Authentication)**, the browser API behind passkeys, security keys, fingerprint login, Face ID, Windows Hello, and FIDO2 authentication.

The flow is:

1. Register a credential (`navigator.credentials.create()`)
2. Authenticate with that credential (`navigator.credentials.get()`)

During registration:

```javascript
navigator.credentials.create(createCredentialDefaultArgs)
```

the browser asks an authenticator (passkey, fingerprint sensor, security key, etc.) to generate:

* A public key
* A private key

The private key stays on the user's device and is never exposed to JavaScript.

The public key and credential ID should be sent to your backend:

```javascript
const credential = await navigator.credentials.create(options);

// Send to server
await fetch("/register", {
  method: "POST",
  body: JSON.stringify({
    id: credential.id,
    rawId: credential.rawId,
    response: credential.response
  })
});
```

During login:

```javascript
navigator.credentials.get(getCredentialDefaultArgs)
```

the authenticator signs a challenge using the stored private key.

The server then verifies the signature using the public key stored during registration.

Important production issues with your example:

❌ Hardcoded challenge

```javascript

```

Challenges must be generated by the server for every request.

Example in Node.js:

```javascript

  id: new Uint8Array(16)
```

G-code (Geometric Code) is the standard programming language used to control CNC machines, 3D printers, laser cutters, milling machines, lathes, and other computer-controlled manufacturing equipment.

Think of it as a set of instructions that tells a machine:

* Where to move
* How fast to move
* When to cut, drill, extrude, or stop
* Which tool to use

For example:

```gcode
G21         ; Use millimeters
G90         ; Absolute positioning
G0 X0 Y0    ; Move quickly to (0,0)
G1 X50 Y50 F1000 ; Move while cutting to (50,50)
M30         ; End program
```
